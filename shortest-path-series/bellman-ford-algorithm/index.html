Bellman-Ford 算法

Bellman-Ford 算法是一种解决包含负权重边的有向图上的单源点最短路径问题的算法。

1. 算法思想

给定有向图 \(G = (V, E)\) 和权重函数 \(w : E \rightarrow \mathbb{R}\)，Bellman-Ford 算法返回一个布尔值以表明是否存在一个从给定源点 \(s\) 可达的负权重环。如果存在这样的环，算法将返回 false。否则，算法将返回 true，并且给出源点 \(s\) 到任意节点的最短路径和它们的权重。

首先，对于每个节点 \(v \in V\)，我们维持一个属性 \(dist[v]\)，即从源点 \(s\) 到 \(v\) 的 最短路径估计值，用来记录从 \(s\) 到 \(v\) 的最短路径权重的上界。我们还需要维持一个属性 \(prev[v]\)，用来记录 \(v\) 在从 \(s\) 到 \(v\) 的最短路径上的 前驱节点。我们使用下面的算法来对最短路径估计值和前驱节点进行初始化，在初始化操作结束后，所有节点的前驱节点都是 \(nil\)（表示 “无”），源点 \(s\) 的最短路径估计值为 \(0\)，其他节点的最短路径估计值为 \(\infty\)。

Bellman-Ford 算法用到了 松弛1 这个概念。对一条边 \((u, v)\) 的松弛过程为：首先测试一下是否可以对从 \(s\) 到 \(v\) 的最短路径进行改善。测试的方法是，将从 \(s\) 到 \(u\) 的最短路径估计值加上从 \(u\) 到 \(v\) 的边权重，与当前的 \(s\) 到 \(v\) 的最短路径估计值进行比较，如果前者更小，则对 \(dist[v]\) 和 \(prev[v]\) 进行更新。通过这种方式，松弛步骤可能降低最短路径的估计值 \(dist[v]\) 并更新 \(v\) 的前驱属性 \(prev[v]\)。

bool relax(int u, int v, double w) {
  if (dist[v] > dist[u] + w) {
    dist[v] = dist[u] + w;
    prev[v] = u;
    return true;
  } else {
    return false;
  }
}

代码说明

在代码中，我们不直接使用权重函数 \(w : E \rightarrow \mathbb{R}\)，而是传入对应边的权重值。

Bellman-Ford 算法通过不断对边进行松驰操作来逐渐地降低从源点 \(s\) 到每个节点 \(v \in V\) 的最短路径估计值 \(dist[v]\)，直到该估计值与实际的最短路径权重 \(\delta(s, v)\) 相同为止。该算法返回 true 当且仅当输入图不存在从源点可达的负权重环。

bool bellman_ford(const Graph &G, int s) {
  initialize(G.n, s);
  for (int i = 0; i < G.n - 1; i++) {
    for (auto [u, v, w] : G.E) {
      relax(u, v, w);
    }
  }
  for (auto [u, v, w] : G.E) {
    if (relax(u, v, w)) {
      return false;
    }
  }
  return true;
}

下图描述的是在有 5 个节点的图上运行 Bellman-Ford 算法的过程。在算法第 2 行对所有节点的 \(dist\) 值和 \(prev\) 值进行初始化后，算法对图的边进行 \(|V| - 1\) 轮处理。每一轮处理对应的是算法第 3–7 行的 for 循环的一轮操作，该循环的每一轮对图的每条边进行一次松驰操作。子图 (b)–(e) 描述的是前 4 轮处理后的算法状态。在进行了 \(|V| - 1\) 轮松驰操作后算法第 8–13 行负责检查输入图中是否存在负权重环并返回与之相适应的布尔值。

Bellman-Ford 算法第 2 行的初始化操作所需时间为 \(O(|V|)\)，第 3–7 行的 for 循环一共要执行 \(|V| - 1\) 轮，每一轮运行时间为 \(O(|E|)\)，第 8–13 行检查的运行时间为 \(O(|E|)\)，因此算法的总运行时间为 \(O(|V| |E|)\)。

2. 正确性证明

回顾最短路径问题的最优子结构，得到最短路径的 传递性质。

传递性质

假设 \(p = \langle s \rightsquigarrow u \rightarrow v\rangle\) 是源点 \(s\) 到节点 \(v\) 的一条最短路径，那么 \(\delta(s, v) = \delta(s, u) + w(u, v)\)。

进一步地，我们有如下 三角形不等式，它与最短路径问题的对偶问题中的约束具有相同的形式。

三角形不等式

对于每条边 \((u, v) \in E\)，总是有 \(\delta(s, v) \le \delta(s, u) + w(u, v)\)。

首先，我们证明在 Bellman-Ford 算法运行过程中，对于任意节点 \(v \in V\)，\(dist[v]\) 总是 \(\delta(s, v)\) 的上界。

上界性质

在 Bellman-Ford 算法运行过程中，对于所有节点 \(v \in V\)，总是有 \(dist[v] \ge \delta(s, v)\)。

证明

使用归纳法进行证明。

  • 基础步：在初始化之后，对于所有节点 \(v \in V\)，有 \(dist[v] = \infty\)，显然 \(dist[v] \ge \delta(s, v)\)；
  • 归纳步：在对边 \((u, v)\) 进行松弛操作时，唯一可能发生变化的值是 \(dist[v] = dist[u] + w(u, v) \ge \delta(s, u) + w(u, v) \ge \delta(s, v)\)。

接着，我们证明松弛操作可以让 \(dist[v]\) 向 \(\delta(s, v)\) 收敛。

收敛性质

假设 \(\langle s \rightsquigarrow u \rightarrow v\rangle\) 是源点 \(s\) 到节点 \(v\) 的一条最短路径，且在对边 \((u, v)\) 进行松弛前有 \(dist[u] = \delta(s, u)\)，那么在松弛后我们将得到 \(dist[v] = \delta(s, v)\)。

证明

显然，在上述条件下，我们有 \(dist[v] \le dist[u] + w(u, v) = \delta(s, u) + w(u, v) = \delta(s, v)\)。因为 \(dist[v]\) 是 \(\delta(s, v)\) 的上界，于是我们得到了 \(dist[v] = \delta(s, v)\)。

最后，我们证明 Bellman-Ford 算法的正确性。

算法正确性

如果输入图中不存在从源点 \(s\) 可达的负权重环，那么 Bellman-Ford 算法将返回 true，且对所有节点 \(v \in V\)，都有 \(dist[v] = \delta(s, v)\)；否则，Bellman-Ford 算法将返回 false。

证明

首先讨论不存在从源点 \(s\) 可达的负权重环的情况。考虑任意节点 \(v \in V\) 和从源点 \(s\) 到 \(v\) 的任意一条最短路径 \(p = \langle v_0, v_1, \dots, v_k\rangle\)，其中 \(v_0 = s\)、\(v_k = v\)。注意，我们只考虑最短简单路径，因此 \(k \le |V| - 1\)。Bellman-Ford 算法的每一轮都会对所有边进行松弛操作，因此边 \((v_{inil}, v_i)\) 一定会在循环的第 \(i\) 轮（\(1 \le i \le k\)）进行松弛。根据收敛性质，在所有松弛操作之后，必然有 \(dist[v] = \delta(s, v)\)。

如果存在从源点 \(s\) 可达的负权重环，那么在 \(|V| - 1\) 轮松弛操作后，必然有节点的 \(dist\) 值未达到其最短路径权重（因为它们的最短路径权重是 \(-\infty\)），还能找到能够成功松弛的边，因此算法的第 10 行将返回 false。

在 Bellman-Ford 算法中，我们执行了 \(|V| - 1\) 轮松弛操作，覆盖了所有可能的最短简单路径。注意，每一轮的松弛操作顺序可以不同，这不影响算法的正确性。

3. 具体实现

思考下面这些问题：

  • Bellman-Ford 算法必须要执行完 \(|V| - 1\) 轮松弛操作才能结束吗？其实只要在某轮循环中发现对所有边进行松弛操作都没能改变任何节点的最短路径估计值，那么算法可以立刻返回 true，因为已经找到了源点到所有节点的最短路径。还有一种特殊的情况，如果在某次松弛后发现 \(dist[s] < 0\)，那么算法可以立刻返回 false，因为算法检测到一个包含源点 \(s\) 的负权重环。
  • 如果 Bellman-Ford 算法返回 true，如何重构最短路径？重构从源点 \(s\) 到给定节点 \(v \in V\) 的最短路径不是难事。如果 \(\delta(s, v) \neq \infty\)，可以利用节点的前驱信息，回溯构造从源点 \(s\) 到节点 \(v \in V\) 的最短路径。否则，不存在从源点 \(s\) 到节点 \(v\) 的路径。
  • 如果 Bellman-Ford 算法返回 false，如何重构负权重环？在 Bellman-Ford 算法中，我们可以在算法返回 false 前，用变量 clue 记录最后一条成功松弛的边的终点。从该节点开始回溯访问前驱节点，直到发现某个节点现了两次，这两次出现位置之间的所有边组成了一个负权重环。注意，变量 clue 所记录的节点可能并不在我们构造出来的负权重环之中。

根据上面的讨论，我们对 Bellman-Ford 算法进行如下改进，并给出了重构最短路径和负权重环的方法。

std::vector<int> reconstruct_negative_weight_cycle() {
  if (clue == nil) {
    return {};
  } else {
    std::vector<int> aux(prev.size(), false);
    std::vector<int> p;
    int x = clue;
    while (not aux[x]) {
      aux[x] = true;
      p.push_back(x);
      x = prev[x];
    }
    std::vector<int> c = {x};
    for (auto it = p.rbegin(); *it != x; it++) {
      c.push_back(*it);
    }
    c.push_back(x);
    return c;
  }
}

事实上，如果输入图中的最短路径最多包含 \(m\) 条边（我们通常无法事先知道这个值），那么 Bellman-Ford 算法最迟会在 \(m + 1\) 轮松弛操作之后结束。在一般情况下，算法可能会更早结束。在 “运气爆棚” 的极限情况下，Bellman-Ford 算法只需要 2 轮循环：第 1 轮，严格按照最短路径的拓扑顺序进行松弛操作，那么这一轮结束后，对所有节点 \(v \in V\) 都有 \(dist[v] = \delta(s, v)\)，接着算法在第 2 轮发现所有松弛操作都失败后，算法直接返回 true。

#include "bellman-ford.h"

int main() {
  const char *names = "stxyz";
  Graph G(5);
  G.add_edge(0, 1, 6);
  G.add_edge(0, 3, 7);
  G.add_edge(1, 2, 5);
  G.add_edge(1, 3, 8);
  G.add_edge(1, 4, -4);
  G.add_edge(2, 1, -2);
  G.add_edge(3, 2, -3);
  G.add_edge(3, 4, 9);
  G.add_edge(4, 0, 2);
  G.add_edge(4, 2, 7);

  BellmanFord BF;
  if (BF.bellman_ford(G, 0)) {
    std::cout << "从源点 " << names[0] << " 出发的最短路径：" << std::endl;
    for (int v = 0; v < G.n; v++) {
      std::vector<int> p = BF.reconstruct_shortest_path(v);
      std::cout << names[v] << " (" << BF.dist[v] << "): ";
      for (int i = 0; i < p.size(); i++) {
        std::cout << names[p[i]] << (i < p.size() - 1 ? "->" : "");
      }
      std::cout << std::endl;
    }
  } else {
    std::cout << "存在从源点 " << names[0] << " 可达的负权重环：";
    std::vector<int> c = BF.reconstruct_negative_weight_cycle();
    for (int i = 0; i < c.size(); i++) {
      std::cout << names[c[i]] << (i < c.size() - 1 ? "->" : "");
    }
    std::cout << std::endl;
  }

  return 0;
}

#ifndef BELLMAN_FORD_H
#define BELLMAN_FORD_H

#include "graph.h"

class BellmanFord {
public:
  std::vector<double> dist;
  std::vector<int> prev;
  int clue;

  bool relax(int u, int v, double w) {
    if (dist[v] > dist[u] + w) {
      dist[v] = dist[u] + w;
      prev[v] = u;
      return true;
    } else {
      return false;
    }
  }

  bool bellman_ford(const Graph &G, int s) {
    dist.assign(G.n, inf);
    prev.assign(G.n, nil);
    dist[s] = 0;
    for (int i = 0; i < G.n - 1; i++) {
      bool relaxed = false;
      for (auto [u, v, w] : G.E) {
        if (relax(u, v, w)) {
          relaxed = true;
          if (dist[s] < 0) {
            clue = s;
            return false;
          }
        }
      }
      if (not relaxed) {
        clue = nil;
        return true;
      }
    }
    for (auto [u, v, w] : G.E) {
      if (relax(u, v, w)) {
        clue = v;
        return false;
      }
    }
    clue = nil;
    return true;
  }

  std::vector<int> reconstruct_shortest_path(int v) {
    if (clue != nil || dist[v] == inf) {
      return {};
    } else {
      std::vector<int> p;
      for (int x = v; x != nil; x = prev[x]) {
        p.push_back(x);
      }
      std::reverse(p.begin(), p.end());
      return p;
    }
  }

  std::vector<int> reconstruct_negative_weight_cycle() {
    if (clue == nil) {
      return {};
    } else {
      std::vector<bool> aux(prev.size(), false);
      std::vector<int> p;
      int x = clue;
      while (not aux[x]) {
        aux[x] = true;
        p.push_back(x);
        x = prev[x];
      }
      std::vector<int> c = {x};
      for (auto it = p.rbegin(); *it != x; it++) {
        c.push_back(*it);
      }
      c.push_back(x);
      return c;
    }
  }
};

#ifndef GRAPH_H
#define GRAPH_H

#include <bits/stdc++.h>

class Graph {
public:
  int n;
  std::vector<std::tuple<int, int, double>> E;
  std::vector<std::vector<std::pair<int, double>>> A;

  Graph(int n) : n(n), E(), A(n) {}

  void add_edge(int u, int v, double w) {
    E.emplace_back(u, v, w);
    A[u].emplace_back(v, w);
  }

  void clear() {
    E.clear();
    A.assign(n, {});
  }
};

const double inf = std::numeric_limits<double>::infinity();
const int nil = -1;

#endif
